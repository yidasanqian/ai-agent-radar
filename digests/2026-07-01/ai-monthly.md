## 一、月度要闻

### 1.1 五大里程碑事件

**🔴 [5月28日] OpenAI Codex Token计数器移除引发社区强烈反弹**

Codex发布rust-v0.135.0 Alpha版本，悄然移除UI中的Token消耗指示器。GitHub Issue #14593单条评论数突破590条，用户指控官方隐瞒成本导致「盲飞」行为。这是本月社区参与度最高的事件，直接推动三大厂商重新审视成本透明度策略。

**🔴 [5月28日] Claude Code Cowork多Agent功能暴露严重稳定性问题**

v2.1.153中Cowork功能在实测中产生高达10GB的VM包体积，并硬性设置100轮对话上限导致长任务强制中断。多个企业用户报告生产环境任务失败，官方承诺重构但尚未给出明确时间表。

**🔴 [6月7日] Claude Code紧急修复Cowork VM崩溃与模型兼容问题**

Anthropic发布v2.1.168紧急修复版，重点解决Cowork VM崩溃与Opus 4.7/4.8 thinking summaries不渲染问题。连续发布6个版本（v2.1.153→v2.1.168），进入高强度修复期。

**🟡 [6月15日] 计费透明度争议达到顶峰**

Claude Code印度INR本地定价Issue获442赞，成为全工具社区参与度最高事件。三大工具成本透明度问题集中爆发：Claude Code用量从80%飙升至100%、OpenAI Codex rate-limit计费暴增10-20倍、Gemini CLI cached/thought tokens遗漏导致成本估算失准。

**🟡 [6月19日] MCP协议企业级能力扩展确立行业标准地位**

三家均推进OAuth认证、Schema验证、通知路由等企业级MCP能力，协议从「工具调用」向「平台间集成标准」演进的趋势明确。Claude Code率先支持`--no-browser`参数实现无头环境认证。

## 三、AI Agent生态月报

### 3.1 MCP协议：从工具调用到平台标准

**本月标志性转变**：MCP（Model Context Protocol）已超越「工具调用协议」定位，演变为三家CLI工具的**平台间集成标准**。

**三家同步推进的企业级能力**：

| 能力维度 | Claude Code | OpenAI Codex | Gemini CLI |
|----------|-------------|--------------|------------|
| **认证体系** | OAuth规范化、--no-browser无头认证 | MCP OAuth序列化栈、跨线程编排 | OAuth刷新原子化、非ASCII header修复 |
| **Schema规范** | 导入路径规范化 | Schema验证 | MIME类型检测、Schema规范化 |
| **通知路由** | MCP工具调用超时机制 | 异步Hook执行 | 通知路由企业级能力 |
| **进程管理** | Hookify规则修复 | 进程池复用 | AST感知文件读取 |

**生态影响评估**：

- MCP正成为CLI工具的「USB-C接口」——跨工具、跨平台互操作性的基础设施
- 企业级需求（SSO、审计日志、权限控制）推动协议能力扩展
- 预计Q3各厂商将发布MCP兼容性认证体系
